ONBOARDING • SEGURANÇA LOCAL

Instalação e Execução em Fluxo Único

Esta página concentra o setup das 3 ferramentas de segurança e o passo a passo para rodar um script único de verificação local. A proposta é reduzir o tempo de configuração e padronizar o início do trabalho de novos desenvolvedores.

Windows

Os comandos abaixo são para Linux/macOS. Se você está no Windows, utilize o WSL 2 (Windows Subsystem for Linux) ou adapte os comandos conforme a documentação oficial de cada ferramenta.

1. Instalação das Ferramentas

1.1 Trivy

Ferramenta de SCA para detectar vulnerabilidades em dependências e arquivos de projeto.

Bash
sudo apt update
sudo apt install -y wget apt-transport-https gnupg lsb-release
wget -qO - https://aquasecurity.github.io/trivy-repo/deb/public.key | gpg --dearmor | sudo tee /usr/share/keyrings/trivy.gpg > /dev/null
echo "deb [signed-by=/usr/share/keyrings/trivy.gpg] https://aquasecurity.github.io/trivy-repo/deb $(lsb_release -sc) main" | sudo tee /etc/apt/sources.list.d/trivy.list
sudo apt update
sudo apt install -y trivy

1.2 Semgrep

Ferramenta de SAST para análise estática de padrões inseguros no código.

Bash
sudo apt update
sudo apt install -y pipx
pipx ensurepath
pipx install semgrep
Autenticação Obrigatória

Semgrep requer login para usar a Pro Engine (regras avançadas). Execute:

Bash
semgrep login

Se preferir usar token (útil em CI), veja a página detalhada do Semgrep.

1.3 Gitleaks

Ferramenta de Secrets Detection para identificar vazamento de credenciais no repositório.

Bash
GITLEAKS_VERSION="8.23.3"
wget -q https://github.com/gitleaks/gitleaks/releases/download/v${GITLEAKS_VERSION}/gitleaks_${GITLEAKS_VERSION}_linux_x64.tar.gz
tar -xzf gitleaks_${GITLEAKS_VERSION}_linux_x64.tar.gz
sudo install -m 755 gitleaks /usr/local/bin/gitleaks
rm -f gitleaks gitleaks_${GITLEAKS_VERSION}_linux_x64.tar.gz LICENSE README.md

2. Validação Rápida

Bash
trivy --version
semgrep --version
gitleaks version

Se os 3 comandos retornarem versão, o ambiente está pronto para execução automatizada.

3. Rodando o Script Pré-definido

3.1 Execução

Criar arquivo e permissao:

Bash
mkdir -p ./scripts
touch ./scripts/security-check.sh
chmod +x ./scripts/security-check.sh

Rodar script:

Bash
./scripts/security-check.sh

Se o arquivo ja existir, pule os dois primeiros comandos e mantenha apenas a edicao do conteudo.

3.2 Frontend (Angular)

Bash
#!/bin/bash

set -euo pipefail

REPORT_MD="relatorio_final.md"
GITLEAKS_JSON="gitleaks_temp.json"
TRIVY_JSON="trivy_temp.json"
SEMGREP_JSON="semgrep_temp.json"

for cmd in jq gitleaks trivy semgrep; do
  if ! command -v "$cmd" > /dev/null 2>&1; then
    echo "Erro: comando '$cmd' nao encontrado."
    exit 1
  fi
done

echo "### Iniciando Varredura de Segurança: $(date) ###"

gitleaks detect -v --report-format json --report-path "$GITLEAKS_JSON" > /dev/null 2>&1
trivy fs ./ --scanners vuln --pkg-types library --format json --output "$TRIVY_JSON" > /dev/null 2>&1
semgrep scan \
  --config=p/typescript \
  --config=p/javascript \
  --config=p/clientside-js \
  --config=p/owasp-top-ten \
  --config=p/security-audit \
  --config=p/cwe-top-25 \
  --severity INFO --severity WARNING --severity ERROR \
  --output="$SEMGREP_JSON" > /dev/null 2>&1

{
    echo "# Relatório Consolidado de Segurança"
    echo "Data: $(date)"
    echo "## 1. Gitleaks (Secrets)"
    if [ -f "$GITLEAKS_JSON" ] && [ -s "$GITLEAKS_JSON" ]; then
        jq -r '.[] | "- **Regra**: \(.Description)\n  **Arquivo**: \(.File):\(.StartLine)"' "$GITLEAKS_JSON"
    else
        echo "✓ Nenhum segredo exposto."
    fi

    echo -e "\n## 2. Trivy (SCA)"
    if [ -f "$TRIVY_JSON" ] && [ -s "$TRIVY_JSON" ]; then
        jq -r '.Results[]? | .Vulnerabilities[]? | "- [\(.Severity)] \(.VulnerabilityID): \(.PkgName)"' "$TRIVY_JSON"
    else
        echo "✓ Nenhuma vulnerabilidade de biblioteca."
    fi

    echo -e "\n## 3. Semgrep (SAST)"
    if [ -f "$SEMGREP_JSON" ] && [ -s "$SEMGREP_JSON" ]; then
        jq -r '.results[] | "- [\(.extra.severity)] \(.check_id): \(.path):\(.start.line)"' "$SEMGREP_JSON"
    else
        echo "✓ Nenhum issue de código."
    fi
} > "$REPORT_MD"

echo "Relatório: $REPORT_MD"

rm -f "$GITLEAKS_JSON" "$TRIVY_JSON" "$SEMGREP_JSON"

4. Resultado Esperado

Exemplo de como deve ficar o arquivo relatorio_final.md ao final da execucao:

Markdown
# Relatório Consolidado de Segurança
Data: Mon Apr 07 16:21:10 -03 2026

## 1. Gitleaks (Secrets)
- **Regra**: GitHub Token
  **Arquivo**: src/environments/environment.ts:22

## 2. Trivy (SCA)
- [HIGH] CVE-2024-12345: axios
- [CRITICAL] CVE-2023-99999: jackson-databind

## 3. Semgrep (SAST)
- [ERROR] java.sql-injection: src/main/java/com/acme/UserController.java:87
- [WARNING] jwt-hardcode: src/main/java/com/acme/security/JwtConfig.java:34
Sem Achados

Quando nao houver ocorrencias, o arquivo deve apresentar as mensagens padrao de sucesso por etapa.

"Nenhum segredo exposto."
"Nenhuma vulnerabilidade de biblioteca."
"Nenhum issue de codigo."
